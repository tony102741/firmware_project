# LLM Review Fields

`src/review/llm_review.py` emits a stable evidence packet for model-based review.

## Core Labels

- `has_rootfs`: did the run reach a real rootfs-style analysis target
- `has_web_ui`: is there evidence for a real web surface, not just a firmware container
- `artifact_kind`: current engine success tier such as `rootfs-success` or `blob-success`
- `probe_readiness`: next-stage readiness such as `rootfs-ready`, `decrypt-probe-ready`, `scan-probe-ready`, or `bundle-probe-ready`
- `blob_family`: blob/container subtype when `artifact_kind` is `blob-success`
- `encrypted_container`: whether the current sample is still opaque ciphertext/container material
- `best_next_action`: the single best immediate follow-up action
- `top_risk_family`: strongest current risk family from the evidence

## Suggested Gold-Label Workflow

1. Generate stubs:
   - `python3 src/review/llm_review.py --corpus research/corpus/firmware_corpus.jsonl --batch-summary runs/regression/batch_regression_summary.json --write-gold-stubs research/review/llm/llm_review_gold.jsonl`
2. Review and correct the autogenerated stubs by hand.
3. Generate review packets:
   - `python3 src/review/llm_review.py --corpus research/corpus/firmware_corpus.jsonl --batch-summary runs/regression/batch_regression_summary.json --emit-corpus-packets research/review/llm/llm_review_packets.jsonl`
   - compact API packets: `python3 src/review/llm_review.py --corpus research/corpus/firmware_corpus.jsonl --batch-summary runs/regression/batch_regression_summary.json --emit-corpus-packets-compact research/review/llm/llm_review_packets_compact.jsonl`
4. Run your model and save one prediction row per `review_id`.
   - local baseline: `python3 src/review/llm_review_infer.py --packets research/review/llm/llm_review_packets.jsonl --provider heuristic --output research/review/llm/llm_review_predictions.jsonl`
   - practical default: `python3 src/review/llm_review_infer.py --packets research/review/llm/llm_review_packets.jsonl --provider hybrid --model gpt-5.2 --preflight --output research/review/llm/llm_review_predictions_hybrid.jsonl`
   - OpenAI API: `python3 src/review/llm_review_infer.py --packets research/review/llm/llm_review_packets.jsonl --provider openai --model gpt-5.2 --output research/review/llm/llm_review_predictions.jsonl`
5. Evaluate:
   - `python3 src/review/llm_review_eval.py --gold research/review/llm/llm_review_gold.jsonl --predictions research/review/llm/llm_review_predictions.jsonl`

## Always-Armed Setup

Create one of these once and the runner will auto-load it on every execution:

- project-local `.env.local` with `OPENAI_API_KEY=...`
- project-local `.secrets/openai_api_key` containing only the key value
- project-local `.env.local` with `ANTHROPIC_API_KEY=...`
- project-local `.secrets/anthropic_api_key` containing only the key value

## Prediction File Shape

Each prediction row can be either of these forms:

```json
{"review_id":"...", "labels":{"has_rootfs":true}}
```

```json
{"review_id":"...", "predictions":{"has_rootfs":true}}
```
